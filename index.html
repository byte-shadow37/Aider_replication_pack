Abstract

Code clones, or duplicated code segments, negatively impact software maintainability, readability, and consistency. Detecting and refactoring code clones is crucial for maintaining high-quality software. Recently, advancements in Artificial Intelligence (AI), particularly Large Language Models (LLMs), have provided new opportunities for enhancing clone detection and refactoring capabilities. Building upon these developments, our paper explores integrating Aider, a command-line AI assistant leveraging LLMs into AntiCopyPaster, an IntelliJ IDEA plugin designed for just-in-time code clone detection and refactoring. The study evaluates the performance of this integration. Our experiments involved testing Aiderâ€™s effectiveness across various clone types and JHotDraw project using five LLMs: Gemini 2.5 Pro, GPT-3.5, GPT-4o, DeepSeek-R1 and Claude 3.7 Sonnet. Results indicate that Gemini 2.5 Pro successfully detected clones effectively. However, for refactoring, 4 cases resulted in incomplete or ineffective refactoring. And there were 13 cases that did not follow the "Extract Method" refactoring. In contrast, GPT-3.5 demonstrated significant limitations, often failing to detect clones or producing unrelated outputs. GPT-4o showed strong capabilities on clone detection and refactoring. It detected all the clones and performed correct refactoring except 6 incomplete method extraction. DeepSeek-R1 detected all the clones, but it only performed 7 correct refactoring, along with 8 incomplete refactoring and 1 ineffective refactoring. It also created multiple new methods in 2 cases, and also in other 2 cases, it directly replace the cloned method with original method call. Claude 3.7 Sonnet successfully refactored 11 cloned. In other 9 cases, there were 6 incomplete refactoring. And it also had one case each for direct replacement with original method call and creating multiple new methods. Additionally, our study evaluated LLMs' capability for recommending meaningful method names for extracted methods. The results revealed that all models generally provided appropriate names. Overall, GPT-4o proposed the most similar names with the original ones. All the names were aligned with Java name conventions and followed camelCase format, and no any other instances of alternative naming formats such as PascalCase, snake\_case, or kebab-case were oberserved. Overall, integrating Aider with advanced LLMs into AntiCopyPaster provides developers choice \eman{new choice is ambiguous} \xing{added utilize LLMs}to utilize LLMs to do code clone detection, refactoring and name suggestion for extracted method.

Private void processFloatLoop(int n, InFunction termFunction, BiConsumer fooCaller) {
    float sum = 0; // C1
    float prod = 1;
    for (int i = 1; i <= n; i++) {

private void calculateLoop(int n, int startValue, OperationOrder order) {
    float sum = 0; // C1
    float prod = 1;
    for (int i = startValue; i <= n; i++) {
        switch(order) {
            case SUM_PROD_FOO:
                sum = sum + i;
                prod = prod * i;
                foo(sum, prod);
                break;
            case PROD_SUM_FOO:
                prod = prod * i;
                sum = sum + i;
                foo(sum, prod);
                break;
            case SUM_FOO_PROD:
                sum = sum + i;
                foo(sum, prod);
                prod = prod * i;
                break;
        }
    }
}

static public int direction(int x1, int y1, int x2, int y2) {
    int direction = 0;
    int vx = x2 - x1;
    int vy = y2 - y1;

    if (vy < vx && vx > -vy) {
        direction = EAST;
    }
    else if (vy > vx && vy > -vx) {
        direction = NORTH;
    }
    else if (vx < vy && vx < -vy) {
        direction = WEST;
    }
    else {
        direction = SOUTH;
    }
    return direction;
}

static public int direction_Cloned(int x1, int y1, int x2, int y2) {
    int d = 0;
    int x = x2 - x1;
    int y = y2 - y1;

    if (y < x && x > -y) {
        d = EAST;
    }
    else if (y > x && y > -x) {
        d = NORTH;
    }
    else if (x < y && x < -y) {
        d = WEST;
    }
    else {
        d = SOUTH;
    }
    return d;
}

Fragmentation of logic example

void buildChildMenus() { 
    JInternalFrame[] array = desktop.getAllFrames();
    updateWindowCommands(array.length > 0);
    
    if (array.length == 0) {
        return;
    }
    
    addSeparator();
    createChildMenuItems(array);
}

private void updateWindowCommands(boolean enabled) {
    cascadeCommand.setEnabled(enabled);
    tileHCommand.setEnabled(enabled);
    tileVCommand.setEnabled(enabled);
    arrangeHCommand.setEnabled(enabled);
    arrangeVCommand.setEnabled(enabled);
}

private void createChildMenuItems(JInternalFrame[] frames) {
    for (int i = 0; i < frames.length; i++) {
        ChildMenuItem menu = new ChildMenuItem(frames[i]);
        menu.setState(i == 0);
        menu.addActionListener(createChildMenuItemListener());
        menu.setIcon(frames[i].getFrameIcon());
        add(menu);
    }
}
